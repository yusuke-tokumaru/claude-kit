# git-issue-create

GitHub または GitLab に Issue を対話的に作成する。

## Phase 1: リモートプロバイダーの検出

```bash
git remote get-url origin
```

URL から自動検出する:
- `github.com` を含む → GitHub（`gh` CLI を使用）
- `gitlab` を含む（セルフホスト含む） → GitLab（`glab` CLI を使用）
- どちらも含まない（Bitbucket 等） → どちらのサービスかユーザーに確認する。gh / glab のどちらでも扱えない場合は対応していない旨を伝えて停止する

リモートが設定されていない場合は、ユーザーに登録先（GitHub / GitLab と `owner/repo` または URL）を確認して進む。

### GitHub の場合: gh CLI の確認

```bash
which gh && gh auth status
```

`gh` が見つからない場合は、公式手順（https://github.com/cli/cli#installation — apt の場合は GitHub の公式リポジトリ追加が必要）でのインストールと `gh auth login` を案内して停止する（パッケージリポジトリの追加や sudo を伴うためスキルからは実行しない）。

### GitLab の場合: glab CLI の確認

```bash
which glab && glab auth status
```

`glab` が見つからない場合は、公式手順（https://gitlab.com/gitlab-org/cli#installation）でのインストールと `glab auth login` を案内して停止する（sudo を伴うためスキルからは実行しない）。

CLI が使えない / 未ログインの場合は案内して停止する。

## Phase 2: イシュー情報の収集

以下の項目をユーザーに確認する（未入力項目はスキップ可）:

| 項目 | 必須 | 説明 |
|------|------|------|
| タイトル | ✅ | 簡潔に問題や要件を表す1行 |
| 種別 | ✅ | `bug` / `feature` / `task` / `その他` |
| 本文 | 推奨 | 下記テンプレートを使用 |
| ラベル | 任意 | カンマ区切りで複数指定可 |
| アサイニー | 任意 | ユーザー名（省略時は自分） |
| マイルストーン | 任意 | マイルストーン名 |

### 本文テンプレート

**バグ報告の場合:**
```
## 概要
（バグの内容を1〜2行で）

## 再現手順
1. 
2. 

## 期待される動作


## 実際の動作


## 環境
- OS: 
- バージョン: 
```

**機能要望・タスクの場合:**
```
## 概要
（やりたいことを1〜2行で）

## 目的・背景


## 詳細・受け入れ条件

```

ユーザーが本文を提供しない場合はテンプレートをそのまま使い、空欄のままで登録する。

## Phase 3: AI 事前診断の生成

コマンド実行前に、コードベースや収集情報を元に AI が事前診断を生成する。

以下の2項目を分析して記述する:

**概要:**
イシューのタイトル・種別・本文をもとに、問題または要件の核心を1〜3文で要約する。

**実装診断:**
コードベースを調査し（必要に応じて `grep` / `find` / `Read` を使う）、以下の観点でまとめる:
- 影響範囲（関連ファイル・モジュール）
- 修正・実装の方針（どこをどう変えれば解決できるか）
- 注意点・リスク（副作用、依存関係など）

生成した診断セクションを本文の末尾に追記する:

```
---
## AI 事前診断

### 概要
（1〜3文の要約）

### 実装診断
**影響範囲:**
（関連ファイル・モジュール）

**方針:**
（修正・実装の方向性）

**注意点:**
（リスク・副作用）

> AIによる事前診断です。タスク実行前に再度確認してください。
```

## Phase 4: イシューの作成

診断を含む本文を確定し、ユーザーに最終確認を求めてからコマンドを実行する。

**GitHub:**
```bash
gh issue create \
  --title "<タイトル>" \
  --body "<本文 + AI事前診断>" \
  [--label "<ラベル>"] \
  [--assignee "<ユーザー名>"] \
  [--milestone "<マイルストーン>"]
```

**GitLab:**
```bash
glab issue create \
  --title "<タイトル>" \
  --description "<本文 + AI事前診断>" \
  [--label "<ラベル>"] \
  [--assignee "<ユーザー名>"]
```

実行後、作成されたイシューの URL を表示してユーザーに知らせる。

## 重要なルール

- Phase を順番通りに実行する。スキップ禁止
- タイトルが空の場合は必ず再入力を求める
- CLI がない / 未ログインの場合は必ず案内で停止する
- AI 事前診断は必ず生成してから本文に追記する（省略禁止）
- Phase 4 の実行前に本文全体（診断含む）をユーザーに提示して確認を得る

## 完了条件

Phase 4 で Issue が作成され、URL をユーザーに表示した時点で完了。
