# ck-todo

やるべきことを `ck todo` CLI で個別 Markdown ファイルとして登録する。日付・スラグ生成・フロントマター（`status: open`）は CLI が処理する。

## 手順

### Step 1: 内容・保存先・優先度を把握する

- `$ARGUMENTS` に TODO 内容が渡された場合はそれを使用する。渡されていない場合は「何をTODOとして登録しますか？」と1行で尋ねる
- `--global` が含まれる、またはユーザーが「全プロジェクト共通で」等と指定した場合のみグローバル保存。それ以外は確認せずローカル
- 優先度: `--priority` 指定、または会話から明らかな場合（「急ぎで」→ high 等）はそれを使う。不明なら medium（CLI のデフォルト）のまま質問しない

### Step 2: 登録する

TODO 内容は動詞から始まる1行に整える。

```bash
ck todo add "<TODO内容>" [--global] [--priority high|medium|low]
```

### Step 3: 完了を報告する

CLI が出力したファイルパスを1行で表示する。

## 制約

- Write / Edit で TODO ファイルを直接作成しない（必ず `ck todo` 経由。フォーマットの源泉は CLI）
- 一覧表示・完了化はこのスキルでは行わない（`/ck-todo-list` の責務）
- `.notes/` は git 管理されるため、実在の顧客名・案件名・認証情報が内容に含まれる場合はそのまま保存せず、先にユーザーに注意を促す

## 完了条件

ファイルパスを報告した時点で完了。
