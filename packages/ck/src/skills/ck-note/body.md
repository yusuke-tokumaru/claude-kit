# ck-note

会話中のメモ・気づき・調査結果を `ck note` CLI で記録する。日付・スラグ生成・フロントマターは CLI が処理する。

## 手順

### Step 1: 内容と保存先を把握する

- `$ARGUMENTS` にメモ内容が渡された場合はそれを使用する。渡されていない場合は「何をメモしますか？」と1行で尋ねる（非対話時は直前の会話文脈から記録すべき内容を要約して使い、要約した旨を報告する。文脈からも特定できなければ何も保存せず停止する）
- `--global` が含まれる、またはユーザーが「全プロジェクト共通で」等と指定した場合のみグローバル保存
- **それ以外は確認せずローカルに保存する**（素早い記録を優先。保存先を質問しない）

### Step 2: 記録する

```bash
ck note "<メモ内容>"            # ローカル: .notes/<date>-<slug>.md
ck note "<メモ内容>" --global   # グローバル: ~/.ck-notes/<date>-<slug>.md
```

同名ファイルが既にある場合は CLI がサフィックス付きのパス（例 `<date>-<slug>-2.md`）で保存する。実際の保存先は CLI の出力で確認する。

### Step 3: 完了を報告する

CLI が出力したファイルパスを1行で表示する。

## 制約

- Write / Edit でノートファイルを直接作成しない（必ず `ck note` 経由。フォーマットの源泉は CLI）
- `.notes/` は git 管理される。内容に実在の顧客名・住所・電話番号・メール・社内システムの認証情報を検知したら、そのまま保存せず伏字化（例: `<顧客A>`・`<TOKEN>`）を提案し、ユーザー承認後にのみ `ck note` で記録する

## 完了条件

ファイルパスを報告した時点で完了。
