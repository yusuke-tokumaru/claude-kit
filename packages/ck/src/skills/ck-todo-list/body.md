# ck-todo-list

登録済みの TODO を `ck todo list` で確認し、ユーザーの指示に従って `ck todo done` で完了化する。

## 手順

### Step 1: 引数を解釈する

- `$ARGUMENTS` に `--global` が含まれる → グローバル（`~/.ck-notes/todos/`）を対象にする
- `--all` が含まれる → done も含めて表示する
- `done <名前>` の形 → Step 2 をスキップして Step 3 へ進む
- 指定なし → ローカルの open のみ（デフォルト）

### Step 2: 一覧を表示する

```bash
ck todo list             # ローカルの open のみ
ck todo list --all       # done も含む
ck todo list --global    # グローバル
```

出力（`[優先度] ファイル名 — 内容`）を以下の表に整形して提示する（`--all` 時は先頭に `[open/done]` が付くため「状態」列を加えた4列にする）:

| 優先度 | ファイル | 内容 |
|--------|---------|------|

- 0件の場合はその旨を1行で伝えて終了する
- ユーザーが「全部見たい」と言った場合はローカルとグローバルの両方を実行して区別して表示する

### Step 3: 完了化する

ユーザーが完了を指示した TODO に対して実行する:

```bash
ck todo done <ファイル名またはスラグの一部>
ck todo done <ファイル名またはスラグの一部> --global   # Step 1 でグローバル対象のとき
```

- **Step 1 で `--global` が指定されていた場合は `done` にも `--global` を付ける**（付けないとローカル側を検索して「見つかりません」になる）。Step 4 の残件数確認（`ck todo list` 再実行）も同様
- 照合は**完全一致を優先し、無ければ部分一致**で解決される。部分一致で複数ヒットしてエラーになった場合は表示された候補を提示し、正確なファイル名で再実行する
- 「すでに完了済み」と出た場合はその旨を伝える
- `status: open が見つかりません` エラーが出た場合、対象ファイルのフロントマターが CLI の想定形式でない（手書き・旧形式）。Write/Edit で直さず、ファイルパスを示して「フロントマターに `status: open` を持つ形式に手動で直してから再実行してください」と案内する

### Step 4: 報告する

完了化したファイルと、残りの open 件数（`ck todo list` を再実行）を1〜2行で報告する。

## 制約

- TODO ファイルを Write / Edit で直接編集しない（必ず `ck todo done` 経由で更新する）
- ユーザーの明示的な指示なしに TODO を done にしない
- TODO の新規登録は `/ck-todo` の責務。このスキルでは行わない

## 完了条件

一覧の提示、または指示された TODO の完了化と残件数の報告が終わった時点で完了。
